# 🤖 AI Telegram News Bot

Автоматический Telegram-бот для публикации IT-новостей с генерацией текста и изображений с помощью AI.

---

## 🚀 Возможности

* 📰 Парсинг новостей из популярных источников (Habr, 3DNews, vc.ru и др.)
* 🧠 Генерация постов через LLM (HuggingFace)
* 🎨 Автоматический подбор изображений (Pexels + fallback AI генерация)
* 💬 Ответы на комментарии с помощью AI
* 🗂 Хранение истории в SQLite
* ⚙️ Автозапуск через GitHub Actions (cron)
* 🧪 Тестовый режим для ручного запуска

---

## 📂 Структура проекта

```
.
├── bot.py                # Основная логика бота
├── bot_memory.db        # SQLite база (создаётся автоматически)
├── last_update_id.txt   # Offset Telegram API
```

## ⚙️ Настройка

### 1. Переменные окружения

Добавь в Secrets:

* `HF_API_TOKEN` — токен HuggingFace
* `TG_BOT_TOKEN` — токен Telegram бота
* `TG_CHAT_ID` — ID канала
* `TG_GROUP_ID` — ID группы (для комментариев)
* `CREATOR_ID` — твой Telegram ID
* `PEXELS_API_KEY` — API ключ Pexels

---

### 2. Запуск

#### Локально:

```bash
python bot.py
```

#### Через GitHub Actions:

* автоматически каждый час
* вручную через `Run workflow`

---

## 🧠 Как работает

1. Бот парсит RSS источники
2. Фильтрует и сортирует новости
3. Генерирует пост через LLM
4. Ищет изображение (Pexels → AI fallback)
5. Публикует в Telegram
6. Сохраняет в БД
7. Отвечает на комментарии

---

## ⚠️ Известные проблемы (важно)

### ❌ 1. `message_id` не сохраняется

Сейчас:

```python
save_post(0, post_content)
```

**Проблема:**

* ответы на комментарии работают некорректно
* бот не знает, к какому посту отвечать

**Как исправить:**
нужно сохранять `message_id` из ответа Telegram API.

---

### ❌ 2. Один offset для всего

Используется один файл:

```
last_update_id.txt
```

**Проблема:**

* конфликт между обработкой команд и комментариев
* возможна потеря сообщений

**Решение:**
разделить на:

```
offset_comments.txt
offset_creator.txt
```

---

### ❌ 3. Кэш базы в CI не работает

```yaml
key: bot-memory-${{ github.run_id }}
```

**Проблема:**

* каждый запуск создаёт новый cache
* база фактически не сохраняется

---

### ❌ 4. Повторное создание AI клиента

Клиент HuggingFace создаётся в каждом цикле.

**Минусы:**

* лишние задержки
* повышенная нагрузка

---

### ❌ 5. Нет retry для запросов

При временной ошибке сети бот падает.
